# 🚨 СРОЧНОЕ ИСПРАВЛЕНИЕ - 503 Service Unavailable

## Проблема
Сервер на Render не запускается (503 ошибка). Скорее всего, он падает при старте из-за отсутствующих колонок в базе данных.

## ✅ Быстрое Решение

### Вариант 1: Добавить миграцию в startup (Рекомендуется)

Нужно добавить миграцию прямо в startup функцию, чтобы колонки создавались при запуске сервера.

#### Шаг 1: Откройте файл main.py
`D:\OdontoHUB\WebApp\Backend-new fix\Backend\app\main.py`

#### Шаг 2: Найдите функцию on_startup()
Она начинается примерно на строке 50:
```python
@app.on_event("startup")
def on_startup():
```

#### Шаг 3: Добавьте миграцию в конец функции
ПЕРЕД последней строкой `Base.metadata.create_all(bind=engine, checkfirst=True)` добавьте:

```python
    # Migrate dentist profile fields if missing
    try:
        from sqlalchemy import text, inspect
        inspector = inspect(engine)
        existing_columns = [col['name'] for col in inspector.get_columns('dentist_profiles')]
        
        fields_to_add = []
        if 'age' not in existing_columns:
            fields_to_add.append(('age', 'INTEGER'))
        if 'experience_years' not in existing_columns:
            fields_to_add.append(('experience_years', 'INTEGER'))
        if 'works_photos' not in existing_columns:
            fields_to_add.append(('works_photos', 'TEXT'))
        
        if fields_to_add:
            print(f"Adding missing dentist profile fields: {[f[0] for f in fields_to_add]}")
            with engine.begin() as conn:
                for field_name, field_type in fields_to_add:
                    conn.execute(text(
                        f"ALTER TABLE dentist_profiles ADD COLUMN {field_name} {field_type}"
                    ))
            print("✓ Dentist profile fields migration completed")
    except Exception as e:
        print(f"Warning: Could not migrate dentist fields: {e}")
        # Don't fail startup if migration fails
```

#### Шаг 4: Сохраните и задеплойте

```powershell

```

#### Шаг 5: Подождите 3-5 минут

Render перезапустит сервер с миграцией.

## Почему это происходит?

1. Сервер пытается запуститься
2. При создании таблиц SQLAlchemy видит, что модель `DentistProfile` имеет поля `age`, `experience_years`, `works_photos`
3. Но в базе данных PostgreSQL эти колонки отсутствуют
4. Сервер падает с ошибкой и не может запуститься
5. Render показывает 503 Service Unavailable
