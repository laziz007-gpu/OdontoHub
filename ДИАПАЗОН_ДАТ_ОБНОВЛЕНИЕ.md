# ✅ ОБНОВЛЕНИЕ: Выбор диапазона дат для выходных дней

## 🎯 Что добавлено

Теперь врачи могут выбирать не только одну дату, но и **диапазон дат** для выходных дней. Это очень удобно для отпусков, длительных поездок или многодневных мероприятий.

## 🔧 Новые возможности

### 1. ✅ Переключатель режимов
- **"Bir kun" (Один день)** - выбор одной даты
- **"Oraliq" (Диапазон)** - выбор диапазона дат

### 2. ✅ Выбор диапазона дат
- **Дата начала** - с какого дня начинается отпуск
- **Дата окончания** - до какого дня включительно
- **Автоматическая валидация** - конечная дата не может быть раньше начальной

### 3. ✅ Предварительный просмотр
- Показывает выбранный период в удобном формате
- Отображает количество дней в диапазоне
- Красивое оформление с подсветкой

### 4. ✅ Умная обработка данных
- Автоматически генерирует все даты между началом и концом
- Отправляет массив дат на бэкенд через bulk API
- Показывает соответствующее уведомление (1 день или N дней)

## 🎨 Как это работает

### Режим "Один день":
1. Врач выбирает одну дату
2. Указывает причину (например: "Личные дела")
3. Нажимает "Qo'shish"
4. Система добавляет один выходной день

### Режим "Диапазон":
1. Врач переключается на режим "Oraliq"
2. Выбирает дату начала (например: 25.12.2024)
3. Выбирает дату окончания (например: 02.01.2025)
4. Видит предварительный просмотр: "25.12.2024 - 02.01.2025 (9 дней)"
5. Указывает причину (например: "Новогодние каникулы")
6. Нажимает "Qo'shish"
7. Система добавляет все 9 дней как выходные

## 🎯 Примеры использования

### 1. Один день (поход в горы):
```
Режим: Bir kun
Дата: 15.01.2025
Причина: Tog'larga sayohat
→ Добавляется 1 выходной день
```

### 2. Отпуск на неделю:
```
Режим: Oraliq
Начало: 01.02.2025
Конец: 07.02.2025
Причина: Oilaviy ta'til
→ Добавляется 7 выходных дней
```

### 3. Длительная командировка:
```
Режим: Oraliq
Начало: 10.03.2025
Конец: 20.03.2025
Причина: Xalqaro konferensiya
→ Добавляется 11 выходных дней
```

## 🔗 Интеграция с бэкендом

Система использует существующий bulk API:
- **Один день**: отправляется массив с одной датой `["2025-01-15"]`
- **Диапазон**: отправляется массив всех дат `["2025-02-01", "2025-02-02", ..., "2025-02-07"]`

Бэкенд обрабатывает оба случая одинаково через эндпоинт `POST /api/days-off/bulk`.
