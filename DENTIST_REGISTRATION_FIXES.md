# Исправления регистрации докторов

## 🎯 Проблемы, которые были решены:

### 1. ❌ Проблема: Доктора автоматически получали статус "approved"
**Решение:** ✅ Теперь новые доктора получают статус "pending" и должны заполнить профиль

### 2. ❌ Проблема: Мок данные создавались автоматически
**Решение:** ✅ Убрали автоматическое создание профилей с фейковыми данными

### 3. ❌ Проблема: Доктора не могли заполнить свои данные
**Решение:** ✅ Создали систему для самостоятельного заполнения профиля

## 🔧 Что было изменено:

### В `auth.py`:
- Изменен статус по умолчанию с `APPROVED` на `PENDING`

### В `dentists.py`:
- Убрано автоматическое создание профилей с мок данными
- Улучшен эндпоинт `/me` - теперь возвращает ошибку если профиль не заполнен
- Улучшен эндпоинт `PUT /me` - теперь доктора могут заполнять свои данные
- Добавлен эндпоинт `GET /pending` - список докторов на одобрение
- Добавлены эндпоинты `/approve` и `/reject` для админов

## 📋 Новый процесс регистрации докторов:

### 1. Регистрация
- Доктор регистрируется с базовой информацией
- Получает статус `pending`
- Получает токен доступа

### 2. Заполнение профиля
- Доктор заходит в свой профиль
- Заполняет обязательные поля:
  - Специализация
  - Клиника
  - ПИНФЛ
  - Номер диплома
- Может заполнить дополнительные поля

### 3. Одобрение админом
- Админ видит список pending докторов
- Проверяет заполненность обязательных полей
- Одобряет или отклоняет заявку
